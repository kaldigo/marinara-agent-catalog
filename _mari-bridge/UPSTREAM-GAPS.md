# Upstream Gaps

This file tracks compatibility helpers that should eventually become upstream
Marinara package APIs.

| ID | Problem | Ideal upstream feature | Current workaround | Packages | Usage count | Risk | Migration notes |
| --- | --- | --- | --- | --- | ---: | --- | --- |
| MB-001 | Capability packages can contribute Conversation Game launchers such as `/uno`, but cannot contribute generic package-owned slash commands to Roleplay or Conversation input. | Client and server package slash command contribution API with help text, autocomplete, command ownership, and mode allowlists. | `_mari-bridge` parses/captures commands before native send and only consumes registered package-owned forms. | presence | 1 | Medium: input DOM/event capture can drift. | Replace bridge capture with official command registration. |
| MB-002 | Summary entry create/generate/edit/toggle/delete events are not exposed to packages. | Stable package event stream for chat summary lifecycle with source metadata. | `_mari-bridge` diffs previous and next summary entry arrays and accepts optional route/SSE hints. | presence | 1 | Medium: inference can miss source details. | Replace diff observer with official summary event callbacks. |
| MB-003 | There is no package hook before Roleplay prompt history is scoped for generation. | Pre-prompt package hook that can mutate or filter prompt history before prompt assembly. | Presence persists native `hiddenFromAICharacterIds` ahead of generation and reconciles on chat changes. | presence | 1 | Low-to-medium: persistence is durable but needs careful ownership metadata. | Move Presence filtering into hook and stop writing temporary visibility state. |
| MB-004 | Built-in chat summaries are globally injected and have no per-character audience. | Per-summary audience controls using character IDs, with prompt scoping matching message history. | Presence mirrors summaries into a chat-scoped lorebook with character filters and disables native summaries. | presence | 1 | Medium: lorebook ordering and chat switch behavior need reconciliation. | Replace mirror with native per-character summary audience fields. |
| MB-005 | Packages cannot add actions to each chat message row. | Message action slot with message ID, chat ID, roster, and update callbacks. | Presence should use native Hide From AI UI where possible and keep any custom UI minimal. | presence | 1 | Medium: DOM actions are fragile if needed. | Replace any DOM action injection with the official slot. |
| MB-006 | Roster change events do not expose a package backfill hook. | Chat roster change event with added/removed character IDs and a transactional message update helper. | Presence stores a roster snapshot and reconciles/backfills when the chat is loaded or updated. | presence | 1 | Low: reconciliation is deterministic but can be delayed. | Move backfill into official roster change callback. |
| MB-007 | Bulk scoped Hide From AI updates are not exposed as a native route. | Bulk update route for `hiddenFromAICharacterIds` that preserves swipe extra consistently. | Presence patches message extra individually or via its own package route. | presence | 1 | Medium: many individual patches can be slow on large ranges. | Replace with native bulk scoped hide route. |
| MB-008 | The package persistence facade can read messages and update chat metadata but cannot patch existing message extra or create/update package-owned lorebooks. | Capability persistence/resources write APIs for message extra and package-owned lorebook + entry reconciliation. | Presence server routes use Fastify `app.inject` against native `/api/chats` and `/api/lorebooks` routes from one isolated helper. | presence | 1 | Medium: route paths and payload schemas are less stable than a capability API. | Replace route injection with first-class package host methods. |

Update `Packages` and `Usage count` when a new package imports a workaround.
